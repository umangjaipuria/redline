Redline

Mark up your agent’s docs like a Google Doc.

Stop dropping back into a chat window to describe which sentence you meant. Your agent writes specs, plans, and PRDs in HTML now. Redline opens the doc in your browser so you can select a sentence and pin a comment to it — then the agent reads your comments, rewrites the doc, and replies back in it. The comments live inside the HTML file. No database, no sidecar, nothing to sync.

Run it

One dependency: Bun ≥ 1.3. Clone, install, point it at any HTML file.

git clone https://github.com/umangjaipuria/redline && cd redline
bun install
bun run start docs/howto.html

It prints a localhost URL — open it. The first run spends a few seconds building the browser UI; every run after that is instant. A file chooser is always a click away in the UI — open another doc any time, or start with no file and Redline opens straight to it.

Or compile it to a binary

Build a standalone binary:

bun run build:binary
./redline your-doc.html

The whole UI is baked into ./redline — no dependencies to install, nothing like Bun, node_modules, or a dist/ folder to keep beside it. Copy it wherever and it just runs.

Set up the Redline skill for your agent

Your agent drives Redline through a skill called redline-review. Symlink it into the skills directory once and never think about it again: 

# Claude Code
ln -s "$(pwd)/.claude/skills/redline-review" ~/.claude/skills/redline-review

# Codex
ln -s "$(pwd)/.agents/skills/redline-review" ~/.agents/skills/redline-review

Some other agent? Point it at this repo’s AGENTS.md — same playbook, no skill needed.

Or just say this

“Clone github.com/umangjaipuria/redline, install the redline-review skill, then open spec.html in Redline for me.”

Comment. Hand it back. Repeat.

Set your name in the top bar so your comments are attributed to you. Then select text — a Comment button shows up (or hit ⌘⇧M / Ctrl⇧M). Write your comment; it anchors to the exact words and drops a card in the margin. Reply to keep a thread going, edit your own messages, delete a thread once it’s handled.

Done leaving comments? Hand it over:

Tell the agent that wrote it

“I left comments in Redline on spec.html. Read them with the redline-review skill, revise the doc, and reply to each thread.”

The agent edits the file directly, replies inline with what it changed, and clears the threads it resolved. Its edits land in your open tab live — no refresh. Read what moved, leave more comments, go again until it’s right.

Agents don’t poll. Each round, tell it to look again — it won’t notice new comments on its own.

Let the agent do the legwork

Most of the setup above is stuff you can just delegate. A few lines worth saying out loud:

Pick one

“Open changelog.html in Redline.”

“Build the redline binary and open this doc in it.”

“Read my Redline comments on the PRD and push back on anything you disagree with before editing.”

Good to know

It stays on your machine

The server binds to 127.0.0.1. Your documents never leave your laptop. You can bind it wider, but that’s off by default — the local API isn’t authenticated, so don’t expose it without thinking.

The comments ride inside the file

No database, no sidecar. Every thread lives in one inert JSON block in the HTML. Commit the file to git, send it to a coworker — the open review travels with it, and the rest of the document stays byte-for-byte what your agent wrote.

Nothing gets silently dropped

Agent reworded the sentence your comment sat on? The comment follows it. Nuked the whole paragraph? It’s flagged needs review or set aside as orphaned — kept in the margin with its original text so you can re-attach it. Comments are never just deleted out from under you.

You review; the agent writes

In this version you comment and the agent owns the text — there’s no direct editing in the page. You point; it types.
