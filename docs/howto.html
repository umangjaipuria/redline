User guide

Getting started with Redline

Redline opens an agent-written HTML document in your browser and lets you comment on it like a Google Doc — select text, leave a note in the margin, reply. Your agent reads the comments, revises the document, and replies inline. The document and all of its review state live in the same file, so there’s nothing extra to sync. This guide takes you from a fresh clone to your first round of review.

1 Install Redline

You need Bun ≥ 1.3 — nothing else. Clone the repo and install:

git clone <repo-url> redline && cd redline
bun install
bun run check   # optional: confirm everything works

2 Set up the Redline skill

Install the redline-review skill so your agent knows how to read your comments and write back. Symlink it into your agent’s skills directory — you only do this once: 

# Claude Code
ln -s "$(pwd)/.claude/skills/redline-review" ~/.claude/skills/redline-review

# Codex
ln -s "$(pwd)/.agents/skills/redline-review" ~/.agents/skills/redline-review

Any other agent can be pointed at this repo’s AGENTS.md, which carries the same guidance.

3 Open a document

Start the server on any HTML file. It prints a localhost URL — open it in your browser.

bun run start path/to/your-doc.html   # open a specific file
bun run start                         # start empty, then open from the UI

The first time you run it, Redline builds the browser UI automatically — a few seconds — then starts instantly after that. You can also run bun run start with no file: Redline opens to a chooser where you click Open a file and browse for a document right in the browser.

One server holds many documents at once. Click the redline logo in the top-left corner at any time to see the list of open documents and switch between them, or open another file.

Redline only reviews existing files

It never creates a document for you — point it at an HTML file your agent already wrote.

4 Leave comments

  1. Type your name in the top bar so your comments are attributed to you.
  2. Select any text in the document — a Comment button appears beside it (or press Cmd+Shift+M, Ctrl+Shift+M on Windows/Linux).
  3. Write your note and post it. The passage is highlighted and a card appears in the margin.
  4. Click a highlight or its card to line the two up. Reply to continue a thread, edit your own messages, or delete a thread once it’s handled.
You’re the reviewer

In this version you comment; your agent owns the document’s text and makes the actual edits. There’s no direct text editing in the page.

5 Hand it to your agent

When you’ve left your comments, tell the agent that wrote the document to review them. With the skill installed it finds the running Redline instance on its own — just say something like:

“I left comments in Redline on your-doc.html. Review them with the redline-review skill, revise the document, and reply to each thread.”

The agent reads the threads, edits the document directly, replies to each one with what it changed, and deletes the threads that are fully resolved. If it can’t locate the server, give it the file path (it can work on the file directly) or the localhost URL the server printed.

6 Iterate

The agent’s edits and replies show up live in your open browser — no refresh needed. Read what changed, leave new comments, and go again until the document is right.

Agents don’t watch for new comments

Each time you finish a round of comments, tell the agent to review again. It won’t notice them on its own.

Good to know

A few things about how Redline works under the hood.

Your comments live in the document

Redline has no database and no sidecar files. Every comment is stored inside a single block in the same HTML file, so the document stays one portable artifact — commit it to git, send it to a colleague, and the open review threads travel with it. Reopen the file later and your comments are still there.

The file barely changes

The only thing Redline ever writes is that one review block (plus a one-line marker so an agent opening the file later knows it’s a Redline document). Highlights are drawn live in the browser, not saved into the file — so apart from the review block, your document’s content stays byte-for-byte what your agent wrote.

Comments are never silently lost

When the agent rewrites text a comment was anchored to, Redline re-finds the passage automatically, even if the wording shifted. If an edit is too drastic to match, the comment is flagged needs review or set aside as orphaned — kept in the margin with its original text so you can re-attach it, never dropped.

Set your name

The name field in the top bar attributes your comments and replies. It’s remembered between sessions, so you set it once. Your agent signs its replies with its own name.

One server, many documents

A single Redline server holds every document you’ve opened. The home button and the title in the top bar take you back to the list to switch between them or open another file.

It’s local and private

The server runs on your own machine and binds to 127.0.0.1 — your documents never leave your computer. (Binding it to a public address is possible but off by default, since the local API isn’t authenticated.)
